# 3D 模型评测指标方法论

## 结论先行

3D 模型评测不要全部交给 VLM。更稳的设计是三层：

1. **可计算指标**：几何、拓扑、复杂度、渲染覆盖率、参考模型距离。便宜、可复现、适合大规模筛查。
2. **VLM 指标**：文本还原度、视觉质量、多视角一致性、材质观感。适合语义和主观质量。
3. **LLM 指标**：把多个指标解释成报告、做 rubric consistency check、生成错误分类和改进建议。不要让纯 LLM 看不到图和 mesh 时直接打 3D 分。

## 项目设计原则

本项目固定采用下面的 trade-off，后续新增指标或前端展示都按这个原则归类：

- **几何客观指标**：用函数直接算，便宜、可复现、适合 scale。
- **语义、视觉、还原度**：用 VLM，看多视角 RGB/Normal。
- **报告、解释、错误归因**：用 LLM 汇总已有指标和 VLM reason，不让纯 LLM 直接凭空打 3D 分。

执行上要保持足够自主性：能由脚本下载、采样、渲染、评测、聚合、可视化的步骤都自动推进；只有涉及不可恢复的破坏性操作、账号授权或外部付费资源时再停下来确认。

## 文献线索

- Gen3DEval: arXiv:2504.08125。用 VLM 做 appearance、surface、text fidelity 成对偏好，再用 ELO 排名。
- GPT-4V Eval for Text-to-3D: arXiv:2401.04092。用 GPT-4V 成对比较 3D 多视角渲染，并用 ELO 聚合，强调和人类偏好对齐。
- T3Bench: arXiv:2310.02977。提出 text-to-3D prompt benchmark，并把 quality 与 text alignment 分开；quality 使用多视角图像分数和视角一致性，alignment 使用 captioning + GPT 评估。
- Eval3D: arXiv:2504.18509。主张细粒度、可解释评测；用多种 foundation model/tool 作为 probes，测语义和几何一致性，而不是只依赖黑盒 VLM。
- Point cloud generation 系列：Achlioptas et al. 2018、PointFlow 等常用 CD、EMD、MMD、Coverage、1-NNA、JSD 来评估点云生成的保真度和多样性。
- ULIP/OpenShape：把 3D、图像、文本对齐到统一 embedding 空间，可用于 zero-shot 分类、text-3D retrieval、语义一致性评估。
- Point-E：公开了 point cloud generation 评测代码思路，包括 P-FID/P-IS、CLIP R-Precision 等。

## 指标分工表

| 指标 | 推荐实现 | 适用问题 | 优点 | 局限 |
| --- | --- | --- | --- | --- |
| vertex_count / face_count | 直接计算 | 模型复杂度、成本 | 极便宜，可规模化 | 不代表质量 |
| bbox_size / aspect_ratio | 直接计算 | 尺寸异常、比例异常 | 稳定 | 需要归一化策略 |
| surface_area / volume | 直接计算 | 几何规模、空洞异常 | 可复现 | 非 watertight 时体积不可靠 |
| watertight | 直接计算 | 是否封闭可打印/可仿真 | 工程价值高 | 生成资产不一定要求封闭 |
| euler_number / components | 直接计算 | 拓扑复杂度、碎片 | 能发现断裂/漂浮 | 对艺术资产解释需谨慎 |
| degenerate_faces / duplicate_faces | 直接计算 | mesh 清洁度 | 可自动化修复 | 不衡量美观 |
| normal consistency | 直接计算/渲染计算 | 法线方向、表面质量 | 和渲染质量相关 | 需要 mesh 法线可靠 |
| Chamfer Distance | 参考模型 + 点采样 | 重建/有 GT 时 | 经典、可复现 | text-to-3D 通常没有唯一 GT |
| EMD | 参考模型 + 点采样 | 点云重建/生成 | 更符合分布搬运 | 计算贵 |
| F-score | 参考模型 + 阈值 | 重建质量 | 易解释 | 阈值敏感 |
| MMD/Coverage/1-NNA/JSD | 生成集 vs 真实集 | 生成分布质量/多样性 | 适合模型级评估 | 需要足够样本和规范采样 |
| CLIP / ImageReward / PickScore | 渲染图 + 文本 | 文本图像对齐、视觉偏好 | 快、便宜 | 2D proxy，可能忽略 3D 问题 |
| ULIP/OpenShape text-3D score | 点云 + 文本 | 3D 语义对齐 | 比纯 2D 更贴近 3D | 接入成本较高 |
| VLM pairwise preference | 多视角图 | 人类偏好、综合质量 | 对齐主观判断 | 成本高，输出需校验 |
| VLM scalar rubric score | 多视角图 | 前端展示、样本筛查 | 直观 | 标定弱于 pairwise |
| LLM report | 表格 + VLM 理由 | 解释、归因、建议 | 用户友好 | 不能替代原始指标 |

## 我们当前 POC 应该实现的指标

### 第一阶段：必须做

- 直接几何：vertex_count、face_count、bbox、surface_area、volume、watertight、components、euler_number。
- 渲染/VLM：appearance、surface_quality、text_fidelity、multi_view_consistency。
- 聚合：按资产、类别、模型、prompt 汇总；支持雷达图/柱状图。

### 第二阶段：有参考模型或多方法输出后做

- Chamfer Distance、F-score、normal consistency。
- 同 prompt 多方法的 pairwise ELO。
- 生成集级别的 MMD、Coverage、1-NNA、JSD。

### 第三阶段：更接近论文和工业评测

- 接 OpenShape/ULIP 做 text-3D retrieval score。
- 接 Eval3D 风格 probes：分割一致性、深度一致性、caption consistency、pixel-level error heatmap。
- LLM 只负责把指标和失败样例整理成中文评测报告。

## 对现有 pipeline 的优化建议

1. **保留 Gen3DEval 的成对偏好**，因为它比直接 1-10 分更稳定。
2. **新增直接计算指标**，作为低成本质量门禁。
3. **前端同时显示“计算指标”和“VLM 指标”**，避免把 VLM 分数误认为客观几何质量。
4. **评测任务分级**：quick = 几何 + 渲染成功率；standard = 几何 + 少量真实 VLM scalar；paper = pairwise + ELO；research = 加 CD/F-score/MMD/COV。
5. **所有 VLM 输出保留 reason 和原始 JSON**，后续用 LLM 做报告，而不是只保存分数。
