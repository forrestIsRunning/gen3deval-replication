# 优化方向

## 1. Python 与依赖管理

使用 `uv` 固定 Python 版本和依赖，避免系统 Python 版本过新导致 3D/科学计算包不兼容。

推荐命令：

```bash
uv python install 3.11
uv venv --python 3.11
uv sync
```

之后所有命令用：

```bash
uv run python scripts/xxx.py
```

## 2. 指标从论文三维扩展到工程六维

论文关注：

- appearance：外观质量；
- surface：表面质量；
- text_fidelity：文本还原度。

为了评测 3D 模型能力，本项目增加：

- geometry_coherence：几何结构是否完整、比例是否合理；
- texture_material：贴图与材质是否清晰、可信；
- multi_view_consistency：多视角是否一致，是否有 Janus 或漂浮部件；
- overall：综合质量。

这些指标更适合作为 3D 模型评测 pipeline 的第一版 POC。

## 3. 两种评测模式

### 单资产评分

输入一个 3D 资产的多视角 RGB/Normal 图，让 VLM 输出 1-10 分。

适合：

- 前端展示；
- 快速对一个数据集跑分；
- 按资产、类别、指标筛查问题。

### 成对偏好 + ELO

输入两个 3D 资产，让 VLM 判断 A/B/tie，然后聚合 ELO。

适合：

- 复现论文主流程；
- 多个 3D 生成方法排名；
- 同 prompt 多方法对比。

## 4. 前端 POC

前端目标不是复杂，而是把评测闭环跑通：

- 选择 VLM；
- 选择 manifest 数据集；
- 设置 limit；
- 触发一轮评分；
- 查看 3D 模型预览；
- 查看多维指标图表；
- 查看每个样本的 VLM 理由。

当前使用 FastAPI + 原生 HTML/JS + `<model-viewer>` + Chart.js，便于理解和修改。
