# 文献综述与指标决策

## 目标

这份文档回答三个问题：

1. arXiv/相关论文里 3D 生成和 3D 模型评测常用哪些指标；
2. 哪些指标应该直接计算，哪些应该交给 VLM，哪些只适合 LLM 汇总；
3. 当前 Gen3DEval 复线 POC 应该怎样优化。

## 常用指标分层

### A. 单资产可计算指标

这些指标不需要 VLM，应该直接用 mesh/point cloud 函数计算：

- `vertex_count`
- `face_count`
- `bbox_x/y/z`
- `aspect_ratio`
- `surface_area`
- `volume`
- `is_watertight`
- `is_winding_consistent`
- `euler_number`
- `degenerate_face_count`
- 可选深度指标：`component_count`、self-intersection、manifoldness

适用场景：

- 大规模质量门禁；
- 找异常模型；
- 控制渲染/推理成本；
- CAD、打印、仿真前检查。

局限：

- 不知道模型是否“像 prompt”；
- 不知道材质是否好看；
- 不知道多视角是否语义一致。

### B. 有参考对象时的几何距离指标

这些指标需要 GT 或参考分布：

- Chamfer Distance, CD
- Earth Mover's Distance, EMD
- F-score
- Normal Consistency
- MMD
- Coverage
- 1-NNA
- JSD

适用场景：

- reconstruction；
- ShapeNet/Objaverse 子集生成；
- 多个生成模型在同一参考分布上比较。

局限：

- text-to-3D 通常没有唯一 GT；
- 点采样和尺度归一化会强烈影响结果；
- CD 可能奖励过平滑结果。

### C. 渲染代理指标

把 3D 渲染成多视角图，再用图像/多模态模型评估：

- CLIP Score
- ImageReward / PickScore
- CLIP R-Precision
- FID/KID on rendered views
- view consistency score

适用场景：

- text alignment；
- 图像观感；
- 快速模型筛选。

局限：

- 本质是 2D proxy；
- 可能漏掉背面 Janus、漂浮碎片、几何破洞；
- 相机、光照、背景影响大。

### D. VLM 指标

VLM 应该看多视角 RGB + normal，输出结构化评分或成对偏好：

- text_fidelity
- appearance
- surface_quality
- geometry_coherence
- texture_material
- multi_view_consistency
- pairwise preference

适用场景：

- 和人类偏好对齐；
- 评估语义还原度；
- 解释失败原因；
- 论文复线的 Gen3DEval/GPT4V-Eval 风格流程。

局限：

- 成本高；
- 输出有随机性；
- scalar score 标定弱，pairwise 更稳；
- 必须保存原始 reason 和 JSON，便于审计。

### E. LLM 指标

纯 LLM 不应该直接给 3D 模型打分。它适合：

- 汇总几何指标 + VLM reason；
- 生成中文报告；
- 做错误分类；
- 给出下一轮数据/模型改进建议；
- 检查评分 rubric 是否一致。

## 推荐指标组合

### Quick Gate

用于 100+ 或 1000+ 资产快速筛查：

- vertex_count
- face_count
- aspect_ratio
- is_watertight
- degenerate_face_count
- render_success_rate

### Standard VLM Eval

用于日常模型比较：

- Quick Gate 全部指标；
- RGB/Normal 多视角渲染；
- VLM scalar scores；
- 每类/每 prompt 聚合；
- 多维图表。

### Paper-Style Eval

用于复线 Gen3DEval：

- 同 prompt 多方法资产；
- appearance / surface / text_fidelity 成对比较；
- ELO 聚合；
- 保留每次比较的 JSON 和 reason。

### Research Eval

用于更接近学术 benchmark：

- CD / EMD / F-score / Normal Consistency；
- MMD / Coverage / 1-NNA / JSD；
- CLIP R-Precision / OpenShape 或 ULIP text-3D score；
- VLM pairwise preference；
- LLM 自动报告。

## 对当前 POC 的优化决策

1. 保持 `compute_geometry_metrics.py` 为默认入口，先覆盖低成本、可规模化指标。
2. 保持 `score_assets.py` 做 VLM scalar score，服务前端和快速筛查。
3. 保持 `evaluate_pairwise.py + aggregate_elo.py` 做论文风格 pairwise ranking。
4. 新增 `analyze_geometry_results.py`，把 120 资产实验变成 CSV、Markdown 报告和图表。
5. 后续优先补 `render_success_rate` 和 `clip_score`，因为它们能连接几何指标和 VLM 判断。
